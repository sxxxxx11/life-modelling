# Life Modelling — 附带作品说明

> 申请岗位：小红书 · 人文训练师实习生

---

## 1. 项目简介

### 做了什么

Life Modelling 是一款免费的人生轨迹推演工具。用户输入当前年龄和 6 大生活模块（睡眠、饮食、运动、工作、休闲、健康）的日常习惯数据后，系统基于马尔可夫链概率模型完成三件事：

1. **推演未来轨迹** — 根据年龄动态调整时间窗口（25 岁以下看 20 年，55 岁以上看 5 年），以折线图展示健康 / 财务 / 关系 / 成长四维度的趋势走向。
2. **锁定核心参数** — 从全部输入中识别出 **1 个** 最能撬动整体人生的变量，并解释其影响链路（如：睡眠 → 次日专注力 → 学习效率 → 成长速度）。
3. **生成微行动方案** — 针对核心参数输出 3 个"轻松到不可能失败"的行动建议，门槛极低（2 分钟内可完成），嵌入已有习惯，不依赖额外意志力。

产品已上线：[sxxxxx11.github.io/life-modelling](https://sxxxxx11.github.io/life-modelling/)，支持中英文切换，所有数据纯本地存储。

### 为什么做

这个项目起源于我在小红书发布的一条视频 ——「用 AI 给人生建模：微调参数、改写底层代码、重启人生」。[视频链接](http://xhslink.com/o/gDagrrBAES)

视频发布后获得 10w+ 浏览量，评论区出现了一个现象：大量观众自发复制视频中展示的 prompt，去豆包等 AI 工具里自行实践，并反馈效果很好。我身边的人也在用。

其中一位使用者的反馈最能概括我做这件事的理由：

> "人是有惰性的。很多事情你知道自己该干，但没有一个外力来 point out 你自己就会一直拖着不去做。但当这个产品出来了，他们可以借助 AI 来明晰自己的未来以及 right now 当下的最小启动应该干什么，就会很有帮助。"

这句话点出了产品的核心价值 —— 它不是又一个鸡汤式的人生导师，而是一个**冷峻但有用的外力**：用数据告诉你"如果什么都不变，未来的你会是什么样"，然后精准指出"现在最值得改变的那一件事是什么"。所有的 prompt 和产品逻辑，都是我和 AI 一路对话聊出来的。

## 2. 使用的工具及与 AI 的协作方式

### 工具链

| 环节 | 工具 |
|------|------|
| 产品构思 & 对话 | 豆包 / 通义千问 等 LLM 对话界面 |
| 全栈开发 | WorkBuddy（AI-native IDE，vibe coding） |
| 前端框架 | 原型阶段用 React + TypeScript + Vite；最终交付为单文件 HTML（零依赖，便于分发） |
| 样式 | Tailwind CSS via CDN |
| 图表渲染 | Canvas API 手写折线图 |
| 数据持久化 | localStorage（纯本地，无需后端） |
| 部署 | GitHub Pages（gh-pages 分支自动部署） |

### 与 AI 的协作方式

整个项目的生命周期可以概括为 **三个阶段的对话循环**：

**阶段一：概念孵化（Prompt Engineering）**

最初我没有写代码的打算。我只是和 AI 聊天 —— 问它"如果把人生当成一组可调参数会是什么样子"、"怎么用马尔可夫链模拟人生轨迹"、"什么样的微行动最容易被执行"。这些对话产出了第一批 prompt，也就是那条小红书视频里的内容。AI 在这里扮演的是**思维伙伴**：帮我结构化模糊直觉、提供跨学科知识（行为科学、概率论、习惯养成理论）、不断追问我的假设是否成立。

**阶段二：产品化落地（Vibe Coding）**

当验证了需求真实存在后，我用 WorkBuddy 开始 vibe coding —— 用自然语言描述需求，让 AI 直接生成代码。具体流程：

1. **设计规格对齐**：先用自然语言写出完整的 SPEC.md（功能定义、交互逻辑、视觉规范、算法思路），AI 基于这份文档理解产品全貌。
2. **增量式构建**：按模块逐步让 AI 实现 —— 先搭骨架（欢迎页 + 步骤导航），再做数据输入层，然后是轨迹推演引擎，最后是微行动生成器。每一步都通过预览确认效果后再继续。
3. **算法调优**：轨迹推演的核心算法（基于马尔可夫链的状态转移概率矩阵）经过多轮调试 —— 让 AI 解释数学原理、生成参考实现、我再根据实际效果反馈调整参数权重。
4. **中英文国际化**：在已有中文版本基础上，让 AI 完成全量翻译并实现语言切换系统，涉及 100+ 个文本键的映射。

在这个阶段，AI 扮演的角色从"对话者"变成了**结对程序员**：它负责语法正确性和工程最佳实践，我负责产品判断和体验决策。

**阶段三：持续迭代**

产品上线后仍在继续迭代。比如评论区用户反馈的某些场景覆盖不足，我会回到 AI 对话中描述问题场景，让它给出改进方案，再用 vibe coding 落地。这个过程是**人主导方向、AI 加速执行**的协作模式。

### 总结

如果用一句话概括我与 AI 的协作方式：**我把 AI 当作思维外骨骼 —— 从最初的灵感碰撞，到中间的代码实现，再到上线后的持续优化，AI 在每个环节降低了"想到"和"做到"之间的摩擦力。而这恰好也是 Life Modelling 这个产品想对它的用户做的事。**
